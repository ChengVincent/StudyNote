### 类的定义
成员变量会自动有初始值，但是局部变量要自己赋初值  
成员变量和局部变量同名，会使用就近原则
### 构造方法
构造方法名和类名相同  
当没有构造方法的时候，系统默认给一个无参构造，当有的时候无论是有参还是无参，系统都不会生成默认。
构造方法可以重载，并且可以在构造方法里面确定初值范围。
### 静态的变量
用static标注的类成员变量为全类共有，通过类直接调用
### 静态方法
静态方法可以直接用类名调用，也可以用类的对象调用，由于静态方法和静态成员变量，类未生成就有，所以，静态方法不能调用非静态成员，以此类推
### 初始化块
```
{} //在勒种直接写成为初始化块，用来初始化类的成员变量
static{} //称为静态初始化块，只能初始化静态类成员变量，初始化块只在类加载时执行一次
//类加载时，先执行静态初始化块，再执行初始化块，最后是构造函数
```
### 类的封装
在实际的操作中，会把成员变量设为private，在利用一个public的成员函数进行取和改值，一遍起名为get，set，这样就不能直接接触到成员变量，很安全
### 包的概念
包是用来区分不同类的文件夹。  
每个包底下的类文件都需要在首行声明package com.chengvincent(包名)  
improt 包名+类名可以引入另一个包中的类，improt 包名+* 导入包下所有的类  
包名全小写，类名大驼峰。

### 用this方法
this代表本类自己
### 内部类
内部类可以提供更好的封装，也就是在一个类里面之窗将另一个类，内部类可以使用任何修饰符，外部类要想访问内部类的内容，先要创建内部类对象，内部类要想访问外部类的对象和方法需要加this关键字，默认不加是使用自己的方法和类，在同名条件下
### 静态内部类
和静态的方法基本一样
### 方法内部类
把类写在另一个类的方法中，也就是说，累不能加访问修饰符和静态标志
### 类的继承
子类可以使用所有父类非private方法
```
class child extends father{
}   //写法
```
###  方法重写
在子类中重写不满足的父类方法，但必须函数名，参数，返回值类型相同，调用时优先调用子类方法。
### 初始化顺序
建立一个子类对象，先初始化父类属性变量，在执行父类初始化函数，在执行子类属性变量，再执行子类初始化函数
### final关键字
修饰类，不能被继承  
修饰方法，不能被重写  
修饰类成员变量，则必须赋初值，不能被默认赋值，  
修饰变量，只能赋值一次，即为常量
### super关键字
在子类中调用父类方法变量要加super  
子类对象默认在初始化时在自己初始化函数第一行（必须）执行父类构造函数super();,也可以自己显式写入，但是必须保证父类有无参构造方法
### Object祖父类
所有的类都是Object的后代类，没有继承的类，默认继承Object，  
toString是object中的一个函数，输出对象的地址哈希，返回值为string，要是我们想输出对象属性，重写他就行了。  
equals方法是用来比较两个对象师傅指向一块内存，重写可以判断是否属性值相同。   
*** 重写equal需要好好看 ***
### 多态
#### 引用多态
父类对象可以指向父类也可以指向子类，但是子类只能指向自己，指向父类会引发未定义行为。  
#### 方法多态
父类指向子类或者父类的话，先调起相应的方法，子类没重写，直接取父类，
### 类型转换
一般情况大类型像小类型做转换没有问题，小类型向大类型转换就会有风险，虽然可以用()做强制类型转换，但是还是会有风险
```
if(a instanceof b)
使用instanceof关键字做a是否可以转换成b的判断，消除隐患
```
### 抽象类
抽象类要求子类必须实现其抽象方法  
抽象类可以有抽象方法和普通方法  
抽象类的引用不能直接用自己的对象，必须指向子类对象
```
public abstract class xxx{
    public abstract function();//抽象方法不需要写函数体
}

abstrct1 a1 = new child()
```
### 接口
```
修饰符 abstract interface 接口名 extends 接口一、接口二
```
由于接口一般是一个类的规范，所以会被继承，修饰符一般是public  
abstract关键字不添加系统也会默认添加   
接口可以多继承  
接口中的变量都是常量默认添加public static final  
函数都是抽象函数，默认public abstract  
```
extend 父类 implements 接口一、接口二
继承接口的方法，继承的父类写在继承接口前
```
接口一般命名前加大写i以示区分  
接口使用和抽象类相同，不能直接用
